常用的mysql语句--入门

2015-04-18
数据库

@列出所有的数据库

mysql> show databases;
+--------------------+
| Database           |
+--------------------+
| information_schema |
| java               |
| kisscss_wp         |
| mysql              |
| performance_schema |
| test               |
+--------------------+

@创建数据库

mysql> create database php;

@删除数据库

mysql> drop database php;

@重命名数据库

@选择数据库

mysql> use php;

@备份数据库

注(mysqldump的路径)：/opt/mysql/bin/mysqldump

mysqldump DBname > backupDBname.sql

@导出结构不导出数据

mysqldump -d 数据库名 -uroot -p > xxx.sql

@导出数据不导出结构

mysqldump -t 数据库名 -uroot -p > xxx.sql

@导出数据和表结构

mysqldump 数据库名 -uroot -p > xxx.sql

@导出特定表的结构

mysqldump -uroot -p -B 数据库名 --table 表名 > xxx.sql

导出整个数据库结构和数据

mysqldump -h localhost -uroot -p123456 database > dump.sql

导出单个数据表结构和数据

mysqldump -h localhost -uroot -p123456  database table > dump.sql

导出整个数据库结构（不包含数据）

mysqldump -h localhost -uroot -p123456  -d database > dump.sql

导出单个数据表结构（不包含数据）

mysqldump -h localhost -uroot -p123456  -d database table > dump.sql

@导入数据库

mysql>use 目标数据库名
mysql>source 导入的文件名
或者：
/opt/mysq/bin/mysql DBname < fileName.sql

注：导出的数据库文件在shell当前路径下(pwd)

mysql>use mysql

mysql> FLUSH PRIVILEGES;

@新建索引

@删除唯一键

mysql> alter table user drop index name;

@删除列

mysql> alter table user drop column age;

mysql> alter table user modify column name varchar(33) character set utf8;

mysql> alter table user change id id int primary key auto_increment;

或者

mysql> alter table user modify column id int(4) primary key auto_increment;
注:以上条sql语句的运行条件是id不是主键。

mysql> alter table user drop primary key,change id id int(5);

truncate table tablename

@给远程访问客户端权限

grant all on *.* to 'username'@'192.168.0.233' identified by 'password'
grant all on *.* to duoli1003a@'192.168.1.%' identified by 'fqd^ui@24wx*'

192.168.0.233:远程ip地址
password:表示远程登陆密码.
然后在233那太电脑上就可以如下方式访问mysql:

@显示某个数据库所有的存储过程名

show procedure status where db='dbname'

@显示存储过程内容

show create procedure proName

@在命令行创建存储过程

注：在命令行一般遇到；就表示语句结束了，所以可以通过改变结束符（delimiter）来避免这个冲突
delimiter //
这样就改变了结束符号为//，可以做如下测试：

mysql> delimiter //
mysql> select * from user;
    -> //
+------+------------+
| id   | name       |
+------+------------+
|    1 | zhangheng  |
|    2 | zhangzhang |
|    3 | hengzhang  |
|    4 | hengheng   |
+------+------------+

说明结束符已经替换成了//，如果想换回来，可以这样：
delimiter ;

@创建存储过程

mysql> create procedure chaxun()
    -> begin
    -> select * from user;
    -> end;
    -> //

@调用存储过程

mysql> call chaxun();
+------+------------+
| id   | name       |
+------+------------+
|    1 | kisscss    |
|    2 | zhangzhang |
|    3 | hengzhang  |
|    4 | hengheng   |
+------+------------+

@删除存储过程

mysql> drop procedure pr_empty;

杀掉指定端口的node进程

2014-09-03
前端开发

通常有多个node进程运行时，需要停掉其中一个，这段代码应该可以帮助你。

#!/bin/bash
if [[ $1 = "" ]];then
    echo "Usage: $0 port"
    exit 8
fi
id=$(netstat -nutlp | grep $1 | awk -F'[ /]+' '{print $7}')
prog=$(netstat -nutlp | grep $1 | awk -F'[ /]+' '{print $8}')
if [ $prog = "node" ];then
    kill $id && echo "stop node is successfully."
else
    echo "not found node process."
fi

Google搜索秘籍

2014-03-11
电脑科学技术 
// add host
127.0.0.1 baidu.com

高级搜索

  • site 表示搜索结果局限于某个具体网站或者网站频道，如163.com，或者是某个域名，如com.cn、com等等。
  • link 语法返回所有链接到某个URL地址的网页
  • inurl 语法返回的网页链接中包含第一个关键字，后面的关键字则出现在链接中或者网页文档中。
  • allinurl 语法返回的网页的链接中包含所有查询关键字。这个查询的对象只集中于网页的链接字符串。
  • allintitle和intitle 的用法类似于上面的allinurl和inurl，只是后者对URL进行查询，而前者对网页的标题栏进行查询。
  • related related用来搜索结构内容方面相似的网页
  • cache 用来搜索GOOGLE服务器上某页面的缓存
  • info 用来显示与某链接相关的一系列搜索，提供cache、link、related和完全包含该链接的网页的功能

以下是实例

1.数值范围：

例如：在豆瓣上查找读过1000本书以上的人
1000.. 本读过 inurl:people site:douban.com
1000.. 人评价 inurl:subject site:book.douban.com

2.文件查找：

例如：查找网上facebook的jpg图片
facebook filetype:jpg
例如：查找关于javascript的PDF格式的电子书
intitle:javascript filetype:PDF
